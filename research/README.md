# Research

Notes here are lightweight, based on public information, and intentionally avoid employer artifacts.

## Tagging (rule)
Every research article should start with a short **hidden tag header** so I can sort/group content later without reorganizing folders.

Use this exact format at the top of each file:

```md
<!--
tags:
  - company:wizard-ai
  - type:overview
  - domain:ecommerce
  - topic:personas
-->
```

### Tag conventions
- `company:<slug>` (e.g., `company:wizard-ai`)
- `type:<overview|deep-dive|notes>`
- `domain:<area>` (e.g., `domain:ecommerce`, `domain:llm-agents`)
- `topic:<topic>` (e.g., `topic:personas`, `topic:multimodal-retrieval`, `topic:product-knowledge`)
- `architecture:<pattern>` (e.g., `architecture:medallion`)
- `technique:<method>` (e.g., `technique:evaluator-agent`, `technique:persona-extraction`)
